Conceptual Models: The Folders That Do Not Exist — 개념적 모델 — 폴더는 실제로 없다
2026년 3월 고1 모의고사 29번 (어법)
🖱 PC: 클릭 → 해석 · 🔊 → 발음
📱 모바일: 탭 → 해석
💡 노란박스 → 쉬운 풀이
일반 문장
문법 핵심
01
A conceptual model is an explanation, usually highly simplified, of how something works.
🔊
개념적 모델은 어떤 것이 어떻게 작동하는지에 대한, 보통 매우 단순화된 설명이다.
📌conceptual개념적인explanation설명simplified단순화된highly매우
✏️ 문법, usually highly simplified, = 삽입 과거분사구 (= which is usually highly simplified), explanation 수식. / of how something works = 전치사 of + 간접의문문(how절), explanation 수식. 간접의문문은 평서문 어순 how S V.
【정의】쉽게 말하면 "무언가가 작동하는 원리를 단순하게 설명한 모델". ★ 이 글의 주제어 — conceptual model 정의.
02
It doesn’t have to be complete or even accurate as long as it is useful.
🔊
그것은 유용하기만 하다면 완전하거나 심지어 정확할 필요도 없다.
📌complete완전한accurate정확한as long as~하기만 하면
✏️ 문법does not have to V = ~할 필요가 없다 (불필요). must not(금지)과 구분! / as long as = 조건 접속사 (~하기만 하면). / complete or even accurate = or로 이어진 형용사 2개 병렬.
03
The files, folders, and icons you see displayed on a computer screen help people create the conceptual model of documents and folders inside the computer, or of apps or applications on the screen.
🔊
컴퓨터 화면에 표시되어 있는 것을 당신이 보는 파일, 폴더, 아이콘은 사람들이 컴퓨터 안의 문서와 폴더에 대한, 또는 화면 위 앱·어플리케이션에 대한 개념적 모델을 만드는 것을 돕는다.
📌displayed표시된create만들다document문서application어플리케이션
✏️ 문법icons (that/which) you see = 목적격 관계사 생략. / see + O + displayed = 지각동사 see + 목적어 + 과거분사(수동 OC). / help + people + create = help + O + 원형부정사(또는 to-V), 여기서 help는 사역적 동사.
【주제문】쉽게 말하면 "화면의 파일·폴더·아이콘 → 우리가 컴퓨터 속에 대한 개념 모델을 만들도록 도와준다". ★ 이 글의 주제.
04
In fact, there are no folders inside the computer—those are effective conceptualizations designed to make them easier to use.
🔊
사실 컴퓨터 안에는 폴더가 없다 — 그것들은 사용하기 더 쉽게 만들도록 고안된 효과적인 개념화이다.
📌effective효과적인conceptualization개념화designed to~하도록 고안된
✏️ 문법designed to V = 과거분사구(수동), conceptualizations 수식 (~하도록 고안된). / make + O + OC: make + them + easier(형용사 비교급)를 OC로 취함. ✗ make them easily(부사는 OC 안 됨) → ✓ make them easier.
【충격 사실】쉽게 말하면 "컴퓨터 안에는 진짜 '폴더'라는 게 없다". 우리가 보는 폴더 아이콘은 쉬운 사용을 위한 개념 장치일 뿐. ★ 반전 — 개념 모델의 본질.
05
Sometimes these depictions can add to the confusion, however.
🔊
그러나 때때로 이러한 묘사가 오히려 혼란을 가중시킬 수도 있다.
📌depiction묘사add to~을 가중시키다confusion혼란
✏️ 문법add to + N = ~을 늘리다, 가중시키다 (자동사 + 전치사, 타동사구). / , however = 부사, 문미 삽입 — 그러나. however 뒤 S+V 아니므로 접속사 아님.
【역접 전환】그런데... 이 편리한 모델이 때로 혼란을 준다. 뒤에 나올 클라우드 이야기의 전조. ★ 흐름 전환 신호.
06
When reading e-mail or visiting a website, the material appears to be on the device, for that is where it is displayed and used.
🔊
이메일을 읽거나 웹사이트를 방문할 때, 그 자료는 기기에 있는 것처럼 보인다, 왜냐하면 그곳이 그것이 표시되고 사용되는 곳이기 때문이다.
📌material자료appear to~처럼 보이다device기기
✏️ 문법When reading ~ or visiting ~ = 분사구문 축약 (= when you read ~). 주절의 주어(the material)가 아니라 일반 you로 해석. / appear to V = ~처럼 보이다. / for = 이유 접속사 (= because). 문두 금지, 주절 뒤에만.
【혼란 예시】쉽게 말하면 "이메일/웹사이트 볼 때 → 내 기기에 자료가 있는 것 같다". 그러나 이게 착각일 수 있음. ★ 다음 문장의 반전을 위한 밑밥.
07
But in fact, in many cases the actual material is “in the cloud,” located on some distant machine.
🔊
그러나 사실, 많은 경우 실제 자료는 '클라우드에' 있고, 어떤 먼 기계에 위치해 있다.
📌actual실제의cloud클라우드distant먼machine기계
✏️ 문법located on ~ = 과거분사구(수동), material 수식 (= which is located). 자료는 '위치되는' 수동 주체. ✗ locating → ✓ located.
【반전】쉽게 말하면 "실제 자료는 내 기기가 아니라 멀리 있는 서버(클라우드)에 있다". 내 화면에 보인다고 내 것이 아님. ★ 개념 모델 vs 실제의 격차.
08
The conceptual model is of one, coherent image, whereas it may actually consist of parts, each located on different machines that could be almost anywhere in the world.
🔊
개념적 모델은 하나의 일관된 이미지에 관한 것이지만, 반면에 그것은 실제로는 여러 부분들로 구성되어 있을 수 있고, 각각은 세상 어디에든 있을 수 있는 서로 다른 기계에 위치해 있다.
📌coherent일관된whereas반면에consist of~로 구성되다
✏️ 문법whereas = 대조 접속사 (~인 반면). / consist of = ~로 구성되다 (수동 형태 be consisted of ✗). / each located ~ = 독립 분사구문 (each가 의미상 주어). / that could be ~ = 주격 관계대명사절, machines 수식.
【결론】쉽게 말하면 "우린 하나의 통일된 이미지를 보지만, 사실 부품들은 전 세계에 흩어져 있다". ★ 개념 모델과 물리적 실체의 괴리 = 이 글의 주제.

문장 01 · 삽입 과거분사구 + of + how절(간접의문문)
A conceptual model is an explanation, usually highly simplified, of how something works.
개념적 모델은 어떤 것이 작동하는 방식에 대한, 보통 매우 단순화된 설명이다.
Tier 1
삽입 과거분사구 (★ 29번 포인트)
, usually highly simplified, = which is usually highly simplified의 관계사+be 생략. explanation을 수식. 설명은 단순화되는 대상 → p.p.(수동). ✗ simplifying → ✓ simplified.
Tier 1
of how절 — 간접의문문
of how something works: 전치사 of + 명사절(간접의문문). 간접의문문은 평서문 어순 (how + S + V). ✗ how does something work → ✓ how something works. what/that와 구분 — 뒤가 완전한 절이면 how/that, 불완전하면 what.
문장 02 · don't have to V (불필요) + as long as (조건)
It does not have to be complete or even accurate as long as it is useful.
그것은 유용하기만 하다면 완전하거나 정확할 필요도 없다.
Tier 1
don't have to vs must not (★ 혼동 포인트)
does not have to V = ~할 필요가 없다(불필요). / must not = ~하면 안 된다(금지). 의미가 정반대이므로 해석 주의. 문맥상 "유용하기만 하면 완전할 필요 없다" → 불필요.
Tier 2
as long as (조건) vs as far as (범위)
as long as = ~하기만 하면 (조건). / as far as = ~하는 한/~까지 (범위, 거리). 여기선 조건이므로 as long as.
Tier 2
병렬 구조 (complete or even accurate)
be동사의 보어 2개를 or로 병렬. 둘 다 형용사로 맞춰야 함.
문장 03 · 관계사 생략 + see + O + p.p. + help + O + 원형
The files, folders, and icons you see displayed on a computer screen help people create the conceptual model of documents and folders.
컴퓨터 화면에 표시되어 보이는 파일, 폴더, 아이콘은 사람들이 개념적 모델을 만드는 것을 돕는다.
Tier 1
목적격 관계대명사 생략 (★)
icons (that/which) you see: see의 목적어 자리인 관계사 생략. 주어(you) + 동사(see) 연결로 관계사 확인 가능.
Tier 1
see + O + 과거분사 (수동 OC, ★ 29번 포인트)
you see [icons] displayed: 지각동사 see + 목적어 + p.p.(수동). 아이콘은 표시되는 수동 대상 → displayed. ✗ displaying(능동) → ✓ displayed(수동).
Tier 1
help + O + 원형 (또는 to-V) (★)
help people create: help는 to가 생략된 원형부정사 또는 to-V를 OC로 취함. to be generated 형태는 불가. ✗ help people created → ✓ help people create (또는 to create).
Tier 2
or of ~ 병렬 구조
of documents ~, or of apps ~: 전치사 of로 시작하는 두 명사구를 or로 병렬. 둘 다 model을 수식.
문장 04 · there are no + 과거분사구 + make + O + 형용사 (★★★ 종합)
In fact, there are no folders inside the computer\u2014those are effective conceptualizations designed to make them easier to use.
사실 컴퓨터 안에는 폴더가 없다 — 그것들은 사용하기 더 쉽게 만들어지도록 고안된 효과적 개념화이다.
Tier 1
designed — 과거분사구 (★)
conceptualizations designed to V: = which are designed to V. 개념화는 고안되는 수동 주체. ✗ designing → ✓ designed.
Tier 1
make + O + 형용사 OC (★★ 29번 빈출)
make them easier: make + 목적어 + 형용사(OC). 비교급(easier)도 형용사. ✗ make them easily(부사는 OC 불가) → ✓ make them easier.
Tier 2
easier to use (형용사 수식 to부정사)
easier to use: to부정사가 형용사(easier)를 수식 — ~하기 쉬운. easy/hard/difficult/simple 같은 형용사는 to부정사와 자주 결합.
문장 05 · add to (자동사 구) + however 삽입
Sometimes these depictions can add to the confusion, however.
그러나 때때로 이러한 묘사가 혼란을 가중시킬 수 있다.
Tier 2
add to + 명사 (자동사구, 의미 주의)
add to N = ~을 증가시키다/가중시키다. add(타동사, ~을 더하다)와 구분. 자동사로 쓰인 구 — 전치사 to 필요.
Tier 2
however는 부사 — 접속사 아님
, however. = 부사로 문미·문중·문두에 삽입. 접속사가 아니므로 문장 연결 기능 없음. (접속사 기능 원하면 But/Yet 사용)
문장 06 · 분사구문 축약 + appear to V + for = because (★★)
When reading e-mail or visiting a website, the material appears to be on the device, for that is where it is displayed and used.
이메일을 읽거나 웹사이트를 방문할 때, 자료는 기기에 있는 것처럼 보인다, 왜냐하면 그곳이 자료가 표시되고 사용되는 곳이기 때문이다.
Tier 1
When reading / visiting — 분사구문 축약
When reading ~ or visiting ~ = When you are reading ~ or visiting ~. 시간/조건 접속사(when/while/if) 뒤의 주어 + be동사 생략. 두 분사가 or로 병렬 → 형태 일치(-ing).
Tier 1
for = because (이유 접속사, ★)
for: 등위접속사로 쓰이면 because와 같은 의미 (이유 보충). 문두 사용 불가 — 반드시 주절 뒤에 콤마와 함께. (※ 문두 전치사 for와 구분).
Tier 2
appear to V (~처럼 보이다)
appears to be: appear + to-V. seem to V와 동일한 쓰임. ✗ appear being → ✓ appear to be.
Tier 2
where 관계부사 (선행사 생략형)
that is where it is displayed: where = the place where. is 뒤 보어 자리에 쓰인 관계부사절. ✗ which(뒤 절이 완전하면 관계부사 필요) → ✓ where.
문장 07 · located 과거분사구 (수동 수식)
But in fact, in many cases the actual material is "in the cloud," located on some distant machine.
그러나 사실 많은 경우 실제 자료는 클라우드에 있고, 어떤 먼 기계에 위치해 있다.
Tier 1
located — 과거분사구 (★ 29번 포인트)
material ~, located on ~: material을 수식하는 과거분사구 (= which is located). 자료는 위치되는 수동 주체 (locate = 위치시키다, 타동사). ✗ locating(능동) → ✓ located.
Tier 3
actual / distant / cloud
actual(실제의, = real) / distant(먼, ant. close) / cloud(클라우드, 원격 서버).
문장 08 · whereas + consist of (수동 불가) + 독립분사구문 + that 관계사 (★★★)
The conceptual model is of one coherent image, whereas it may actually consist of parts, each located on different machines that could be almost anywhere.
개념 모델은 하나의 일관된 이미지이지만, 반면에 실제로는 여러 부분들로 구성될 수 있고, 각각은 세상 어디에든 있을 수 있는 다른 기계에 위치해 있다.
Tier 1
whereas (대조 접속사)
whereas = 반면에 (대조). = while. 앞·뒤가 대조되는 두 절을 연결. 의미: 하나의 이미지 ↔ 흩어진 부품들.
Tier 1
consist of — 수동태 불가 (★ 빈출 오류)
consist of: 자동사구로 수동태 불가. ✗ be consisted of → ✓ consist of. 같은 계열: resemble, belong to, appear(나타나다) 등.
Tier 1
each located ~ (독립분사구문)
each located on ~: 독립분사구문 (= and each is located on ~). each가 의미상 주어. 부분들 각각이 위치되는 수동 → located(p.p.).
Tier 1
that — 주격 관계대명사
machines that could be ~: 선행사 machines(사물) + 관계사 자리에 주어 없음 → 주격. that / which 가능. ✗ what(선행사 포함) → ✓ that.
Tier 3
coherent / consist of / whereas
핵심 어휘 3종: coherent(일관된) / consist of(~로 구성되다) / whereas(반면에).
