Conceptual Models — 우리는 실제가 아닌 모델로 세상을 이해한다
2026년 3월 고1 모의고사 · 폴더·아이콘·클라우드의 진실
🖱 PC: 마우스 올리기 → 해석 · 클릭 → 발음
📱 모바일: 탭 → 해석 · 🔊 → 발음
💡 노란박스 → 쉬운 풀이
일반 문장
문법 핵심
01
A conceptual model is an explanation, usually highly simplified, of how something works.
🔊
개념 모델은 무언가가 어떻게 작동하는지에 대한 설명으로, 보통 매우 단순화된 것입니다.
📌conceptual model개념 모델explanation설명highly simplified매우 단순화된
✏️ 문법usually highly simplified = 삽입 분사구 (쉼표로 삽입) / how something works = 간접의문문 (의문사 + 주어 + 동사)
💡 쉬운 풀이
'conceptual model' — 실제 작동 방식이 아닌 이해를 돕기 위한 단순화된 설명이에요.
★ 진짜 구조는 복잡하지만, 쉽게 이해할 수 있도록 만든 비유나 그림 같은 것!
02
It doesn't have to be complete or even accurate as long as it is useful.
🔊
(= 개념 모델)그것은 유용하기만 하다면 완전하거나 심지어 정확할 필요도 없습니다.
📌accurate정확한as long as~하기만 하다면useful유용한
✏️ 문법doesn't have to be = ~일 필요가 없다 / or even = 심지어 ~도 / as long as = ~하기만 하다면 (조건)
💡 쉬운 풀이
정확하지 않아도 된다는 게 핵심이에요!
★ '틀려도 괜찮다 — 내가 사용하는 데 도움이 되면 그걸로 충분하다!'
03
The files, folders, and icons you see displayed on a computer screen help people create the conceptual model of documents and folders inside the computer, or of apps or applications on the screen.
🔊
컴퓨터 화면에 표시된 파일, 폴더, 아이콘들은 사람들이 컴퓨터 내부의 문서와 폴더, 또는 화면의 앱이나 응용 프로그램에 대한 개념 모델을 만드는 데 도움을 줍니다.
📌displayed표시된application응용 프로그램create a model모델을 만들다
✏️ 문법you see displayed = 목적격 관계대명사 생략 + 5형식 (see + 목적어 + 과거분사) / help people create = help + 목적어 + 동사원형
💡 쉬운 풀이
'icons help people create the conceptual model' — 폴더 아이콘이 실제 폴더가 아니라 개념 모델을 만들어주는 도구예요.
★ 우리는 화면의 아이콘을 보며 컴퓨터 내부를 이해한다 — 실제와 달라도 이해에 도움이 되면 된다!
04
In fact, there are no folders inside the computer — those are effective conceptualizations designed to make them easier to use.
🔊
사실, 컴퓨터 내부에는 폴더가 없습니다 — 그것들은(= 폴더 아이콘들) 사용을 더 쉽게 만들기 위해 설계된 효과적인 개념화입니다.
📌conceptualization개념화designed to~하도록 설계된easier to use사용하기 더 쉬운
✏️ 문법In fact = 사실 / there are no folders = 폴더가 없다 / designed to make them easier to use = 과거분사 후치수식 + 5형식 make
💡 쉬운 풀이
폴더는 실제로 존재하지 않아요!
★ 컴퓨터 안에는 전기 신호만 있을 뿐 — 폴더는 이해하기 쉽도록 만든 허구의 모델!
05
Sometimes these depictions can add to the confusion, however.
🔊
하지만 때때로 이러한 묘사들은(= 폴더·아이콘 등의 개념화) 혼란을 더할 수 있습니다.
📌depiction묘사, 표현add to~을 더하다, 가중시키다confusion혼란
✏️ 문법add to = ~을 더하다, 증가시키다 / however = 하지만 (역접 접속부사, 문장 끝 위치)
💡 쉬운 풀이
개념 모델이 오히려 혼란을 만들 수 있다는 전환점이에요.
★ 이해를 돕기 위해 만든 것이 오히려 잘못된 이해를 만들기도 한다!
06
When reading e-mail or visiting a website, the material appears to be on the device, for that is where it is displayed and used.
🔊
이메일을 읽거나 웹사이트를 방문할 때, 그 자료는 기기 위에 있는 것처럼 보입니다. 왜냐하면 그곳이 바로 자료가 표시되고 사용되는 곳이기 때문입니다.
📌material자료, 내용물appear to be~인 것처럼 보이다device기기, 장치
✏️ 문법appears to be = ~인 것처럼 보이다 / for = 왜냐하면 (이유 접속사) / that is where = 그곳이 바로 ~하는 곳이다
💡 쉬운 풀이
이메일이나 웹사이트 자료가 내 기기에 있는 것처럼 보이지만 — 실제로는?
★ for(왜냐하면) = 이유를 설명하는 접속사. 그곳이 표시되고 사용되는 곳이니까 거기 있는 것처럼 느끼는 것!
07
But in fact, in many cases the actual material is "in the cloud," located on some distant machine.
🔊
하지만 사실, 많은 경우에 실제 자료는 '클라우드에' 있으며, 어떤 먼 곳의 기계에 위치합니다.
📌actual실제의in the cloud클라우드에 (인터넷 서버에)distant먼, 멀리 있는
✏️ 문법But in fact = 하지만 사실 / located on = ~에 위치한 (분사구문, is in the cloud의 부연 설명)
💡 쉬운 풀이
'in the cloud' — 구름 속이 아니라 인터넷 어딘가의 서버에 있다는 뜻이에요.
★ 내 폰에 있는 것 같지만 사실 지구 반대편 데이터 센터에 있다 — 클라우드는 개념 모델의 대표적 예!
08
The conceptual model is of one, coherent image, whereas it may actually consist of parts, each located on different machines that could be almost anywhere in the world.
🔊
개념 모델은 하나의 일관된 이미지이지만, 실제로는 그것이(= 클라우드의 자료가) 세계 어디에나 있을 수 있는 서로 다른 기계들에 각각 위치한 여러 부분으로 구성될 수 있습니다.
📌coherent일관된, 통일된whereas~인 반면에consist of~로 구성되다
✏️ 문법whereas = ~인 반면에 (강한 대조 접속사) / consist of = ~로 구성되다 / each located on = 각각 ~에 위치한 (분사구문)
💡 쉬운 풀이
'one coherent image whereas ~ consist of parts' — whereas(반면에)가 핵심 대조예요.
【개념 모델】 하나의 파일처럼 보임
【실제 현실】 전 세계 여러 서버에 분산된 조각들
★ 개념 모델과 현실의 간극 — 이것이 혼란을 만드는 원인!
📖 줄거리 & 핵심 정리

💡 주제 한 줄 요약

개념 모델은 정확하지 않아도 된다 — 폴더는 실제로 없고, 클라우드 파일은 전 세계 서버에 흩어져 있다!

Conceptual models don't need to be accurate — just useful. Folders don't exist; cloud files are scattered across distant machines.

🧠 개념 모델이란?
  • 단순화된 설명 — 정확할 필요 없음
  • 유용(useful)하기만 하면 충분!
  • 예시: 컴퓨터 화면의 파일·폴더·아이콘
  • 사실 컴퓨터 안에 폴더는 없다!
☁️ 클라우드의 진실
  • 이메일·웹: 기기에 있는 것처럼 보임
  • 실제: 먼 곳의 서버(distant machine)에!
  • 개념 모델 = 하나의 coherent 이미지
  • 현실 = 전 세계 여러 서버의 조각들
⚠️ 혼란의 함정
  • depictions → add to the confusion
  • 이해를 돕기도, 잘못된 이해도 만듦
  • '내 기기에 있다'는 착각
  • → 보안·프라이버시 오해 야기!
항목개념 모델 (보이는 것)실제 현실
컴퓨터 폴더화면의 폴더 아이콘실제 폴더 없음 — 전기 신호만
클라우드 파일내 기기에 있는 것처럼전 세계 먼 서버에 위치
구성 방식하나의 coherent 이미지여러 조각이 다른 기계에 분산
정확성정확하지 않아도 됨실제 구조는 훨씬 복잡
